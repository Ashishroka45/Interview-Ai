# Interview-AI
🚀 **Live Demo:** https://interview-ai-dfa2.vercel.app/
AI-powered interview preparation platform that turns a resume and a job description into a personalized mock interview report — technical and behavioral questions, a skill-gap analysis, and a day-by-day preparation plan, all generated by Google Gemini and exportable as a PDF.

## Overview

Interview-AI is a full-stack MERN application built around one core workflow: upload your resume (PDF), paste the job description you're targeting, and optionally add a short self-description. The backend parses your resume, sends it to Gemini alongside the job description, and returns a structured interview report containing:

- **Match score** — how well your profile fits the job description
- **Technical questions** — with model answers and the interviewer's intent behind each one
- **Behavioral questions** — with STAR-based sample answers
- **Skill gap analysis** — missing or weak skills, ranked by severity
- **Preparation plan** — a day-by-day study schedule leading up to the interview

Reports are saved to your account, can be revisited later, and can be exported as a downloadable PDF.

## Features

- 🔐 **JWT-based authentication** — register/login with hashed passwords (bcrypt), HTTP-only cookie sessions, and token blacklisting on logout
- 📄 **Resume parsing** — extracts text from uploaded PDF resumes server-side
- 🤖 **AI-generated interview reports** — powered by Gemini 2.5 Flash with a strict, schema-validated (Zod) JSON response
- 📊 **Skill gap & prep plan** — actionable, prioritized feedback instead of just a list of questions
- 🧾 **PDF export** — generate a downloadable report via Puppeteer
- 📁 **Report history** — paginated list of past interview reports per user
- ⚛️ **React 19 SPA frontend** — feature-based structure with protected routes, toast notifications, loading/skeleton states, and reusable UI primitives

## Tech Stack

**Frontend (`Client/`)**
- React 19 + Vite
- React Router
- React Hook Form
- Axios
- SCSS (Sass)
- ESLint

**Backend (`Server/`)**
- Node.js + Express 5
- MongoDB + Mongoose
- Google Gemini API (`@google/genai`)
- JWT (`jsonwebtoken`) + `bcryptjs` for auth
- `multer` for file uploads, `pdf-parse` for resume text extraction
- `puppeteer` for PDF report generation
- `zod` for AI response schema validation
- `cookie-parser`, `cors`, `dotenv`, `nodemon`

## Getting Started

### Prerequisites

- Node.js (v18+ recommended)
- MongoDB (local instance or Atlas connection string)
- A Google Gemini API key

### 1. Clone the repository

```bash
git clone https://github.com/Ashishroka45/Interview-Ai.git
cd Interview-Ai
```

### 2. Set up the backend

```bash
cd Server
npm install
```

Create a `.env` file in `Server/` with the following variables:

```env
PORT=8000
MONGODB_URL=your_mongodb_connection_string
JWT_SECRET=your_jwt_secret
GEMINI_API_KEY=your_gemini_api_key
NODE_ENV=development
```

Start the server:

```bash
npm run dev
```

The API will run on `http://localhost:8000` by default.

### 3. Set up the frontend

```bash
cd ../Client
npm install
npm run dev
```

The client will run on `http://localhost:5173` by default (as expected by the server's CORS configuration).

## API Reference

### Auth — `/api/auth`

| Method | Endpoint    | Description                  | Access  |
|--------|-------------|-------------------------------|---------|
| POST   | `/register` | Register a new user           | Public  |
| POST   | `/login`    | Log in an existing user       | Public  |
| GET    | `/logout`   | Log out (blacklists token)    | Private |
| GET    | `/getMe`    | Get the currently logged-in user | Private |

### Interviews — `/api/interview`

| Method | Endpoint                       | Description                                            | Access  |
|--------|--------------------------------|---------------------------------------------------------|---------|
| POST   | `/`                             | Upload a resume + job description to generate a report | Private |
| GET    | `/report/:interviewId`          | Fetch a single interview report                         | Private |
| GET    | `/reports`                      | Fetch a paginated list of the user's reports            | Private |
| POST   | `/generatepdf/:interviewId`     | Generate and download the report as a PDF               | Private |
